# ADRs

This directory stores Architectural Decision Records for `tz-player`.

## Naming

Use `ADR-XXXX-short-title.md` where `XXXX` is a zero-padded sequence, e.g. `ADR-0001-incremental-stabilization.md`.

## Lifecycle States

- `Proposed`
- `Accepted`
- `Superseded`
- `Deprecated`

## When to write an ADR

Write or update an ADR for decisions that affect:

- keyboard/focus interaction model
- data model or persistence contract
- playback backend architecture
- major testing strategy changes
- dependency additions with architectural impact
